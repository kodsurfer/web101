# Интерактивные механики с курсором и случайностью в веб-дизайне

## Введение
Сегодня мы погрузимся в мир интерактивности и научимся оживлять наши макеты с помощью JavaScript. Мы разберём, как заставить элементы интерфейса реагировать на движение мыши, и добавим немного случайности, чтобы создать эффект живого, дышащего дизайна. Воркшоп построен так, чтобы вы не просто скопировали код, а поняли логику и смогли применить её в своих проектах.

**Цели воркшопа:**
- Понять, как отслеживать положение курсора и вычислять направление к элементу.
- Освоить базовые математические методы (`Math.atan2`, `Math.PI`) и свойства (`offsetLeft`, `offsetTop`, `pageX`).
- Научиться применять CSS-трансформации через JavaScript для создания визуального отклика.
- Поэкспериментировать со случайными значениями (`Math.random`), чтобы добавить непредсказуемости.

## Теория

### Как браузер сообщает нам о движении мыши?
Когда пользователь двигает мышью, браузер генерирует события, которые мы можем «поймать» и обработать. Основное событие – `mousemove`. Мы можем слушать его на любом элементе или на всём документе.

```javascript
element.addEventListener('mousemove', function(event) {
  // здесь у нас есть объект event с данными о положении мыши
});
```

Объект `event` содержит координаты курсора:
- `event.pageX`, `event.pageY` – координаты относительно всего документа (включая прокрутку).
- `event.clientX`, `event.clientY` – координаты относительно видимой области окна.

Для наших целей удобно использовать `pageX` и `pageY`, чтобы не зависеть от прокрутки.

### Как найти центр элемента?
Чтобы элемент «смотрел» на курсор, нам нужно знать его центр. Для этого используем свойства элемента:
- `offsetLeft` – расстояние от левого края родителя до левого края элемента.
- `offsetTop` – расстояние от верхнего края родителя до верхнего края элемента.
- `offsetWidth`, `offsetHeight` – ширина и высота элемента.

Центр по горизонтали: `offsetLeft + offsetWidth / 2`  
Центр по вертикали: `offsetTop + offsetHeight / 2`

### Откуда берётся угол?
Нам нужно вычислить угол между направлением от центра элемента к курсору и вертикальной осью (или горизонтальной, в зависимости от того, как мы хотим вращать). Здесь на помощь приходит арктангенс.

В JavaScript есть метод `Math.atan2(y, x)`, который возвращает угол в радианах между положительной осью X и точкой (x, y). Но мы хотим угол относительно вертикали, поэтому передадим разницу координат особым образом.

В коде, который мы разберём, используется:
```javascript
let rad = Math.atan2(event.pageX - x, event.pageY - y);
```
Здесь:
- `event.pageX - x` – смещение по горизонтали.
- `event.pageY - y` – смещение по вертикали.

`Math.atan2(dx, dy)` даёт угол, где 0 радиан соответствует направлению **вверх** (положительная ось Y). Это удобно для вращения элемента, который по умолчанию смотрит вверх.

### Перевод радиан в градусы и поворот
CSS-свойство `transform: rotate()` принимает градусы. Формула перевода:  
`градусы = (180 / Math.PI) * радианы`

В коде мы видим:
```javascript
let rot = (180 / Math.PI) * rad * -1;
```
Умножение на `-1` нужно, чтобы изменить направление вращения (обычно положительный угол – по часовой стрелке, но в зависимости от ориентации элемента может потребоваться инверсия).

### Собираем всё вместе: следим за курсором
Мы навешиваем обработчик `mousemove` на контейнер (например, с классом `moveArea`). Внутри находим все элементы с классом `eye` и для каждого вычисляем угол и применяем поворот.

## Разбор готового кода
Давайте посмотрим на код, который реализует эффект следящих глаз.

```javascript
document.addEventListener('DOMContentLoaded', () => {
  mouseMoveEyes();
});

function mouseMoveEyes() {
  let moveArea = document.querySelector('.moveArea');

  moveArea.addEventListener('mousemove', () => {
    let eyes = document.querySelectorAll('.eye');

    eyes.forEach((eye) => {
      let x = eye.offsetWidth / 2 + eye.offsetLeft;
      let y = eye.offsetHeight / 2 + eye.offsetTop;
      let rad = Math.atan2(event.pageX - x, event.pageY - y);
      let rot = (180 / Math.PI) * rad * -1;

      eye.style = `transform: rotate(${rot}deg)`;
    });
  });
}
```

**Пояснения:**
- `DOMContentLoaded` гарантирует, что скрипт выполнится после полной загрузки HTML-структуры.
- `mouseMoveEyes` ищет область `.moveArea` и вешает на неё обработчик движения мыши.
- При движении мыши перебираются все элементы `.eye`.
- Для каждого глаза вычисляются координаты центра.
- `event` – это неявный параметр, который передаётся в функцию обработчика; он доступен как глобальная переменная внутри обработчика (но лучше писать явно `(event) =>`).
- Вычисляется разница координат и угол.
- Угол преобразуется в градусы и инвертируется.
- Применяется CSS-трансформация.

### Обратите внимание на важные моменты:
- Координаты мыши берутся относительно документа (`pageX`, `pageY`), а координаты элемента – относительно позиционированного родителя. Это работает, если контейнер `.moveArea` не имеет прокрутки и его левый верхний угол совпадает с началом документа. В противном случае могут быть смещения. Для более точного расчёта можно использовать `getBoundingClientRect()` и координаты `clientX/clientY`.
- `event` должен быть передан явно: `addEventListener('mousemove', (event) => {...})`. В исходном коде это опущено, но мы будем писать правильно.

Создайте простую страницу с двумя «глазами» – кружками с зрачками. Каждый глаз – это контейнер с классом `eye`. Внутри можно разместить круглый зрачок, но для начала мы будем вращать весь глаз. Главное, чтобы у глаза была явная форма (background, border-radius), чтобы поворот был заметен.

```css
.moveArea {
```

### Задание 2. Подключаем JavaScript
Вставьте приведённый выше код в ваш файл script.js, исправив явную передачу `event`:

```javascript
document.addEventListener('DOMContentLoaded', () => {

function mouseMoveEyes() {
  const moveArea = document.querySelector('.moveArea');

  moveArea.addEventListener('mousemove', (event) => {

      const y = eye.offsetTop + eye.offsetHeight / 2;
```

Проверьте результат: при движении мыши в пределах `.moveArea` глаза должны поворачиваться в сторону курсора. Если вращение идёт не туда, поэкспериментируйте со знаком (`* -1`).

### Задание 3. Улучшаем точность с помощью `getBoundingClientRect()`
Как мы заметили, `offsetLeft` и `offsetTop` считают относительно родителя. Если родитель не привязан к началу координат документа, расчёты будут неверны. Давайте используем более надёжный метод – `getBoundingClientRect()`. Он возвращает координаты элемента относительно окна, но нам нужно будет согласовать их с координатами мыши (`clientX/clientY`), чтобы обе величины были в одной системе.

```javascript
moveArea.addEventListener('mousemove', (event) => {
  const eyes = document.querySelectorAll('.eye');
  eyes.forEach((eye) => {
    const rect = eye.getBoundingClientRect();
    const x = rect.left + rect.width / 2;
    const y = rect.top + rect.height / 2;
    const rad = Math.atan2(event.clientX - x, event.clientY - y);
    const rot = rad * 180 / Math.PI * -1;
    eye.style.transform = `rotate(${rot}deg)`;
  });
});
```

Обновите код и проверьте – теперь вращение должно работать независимо от позиции контейнера на странице.

### Задание 4. Добавляем случайность (рандом)
Теперь, когда базовая механика готова, давайте добавим элемент непредсказуемости. Пусть глаза не только следят за курсором, но и слегка дрожат или меняют размер зрачка случайным образом.

**Идея:** при каждом движении мыши, помимо поворота, будем немного смещать зрачок (псевдоэлемент) с помощью `transform: translate()` со случайными значениями.

Для этого нам нужно получить псевдоэлемент? Но напрямую через JS их стилизовать сложно. Проще добавить внутрь каждого `.eye` отдельный элемент-зрачок (например, `<div class="pupil"></div>`). Тогда мы сможем управлять им через JS.

**Изменения в HTML:**
```html
<div class="eye">
  <div class="pupil"></div>
</div>
```
**CSS:**
```css
.eye {
  /* ... */
  position: relative;
}
.pupil {
  width: 30px;
  height: 30px;
  background: #000;
  border-radius: 50%;
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  transition: transform 0.05s; /* для плавности */
}
```
Теперь зрачок позиционирован по центру глаза. Мы можем менять его `transform` дополнительно, чтобы он смещался относительно центра.

**JavaScript с рандомом:**
```javascript
moveArea.addEventListener('mousemove', (event) => {
  const eyes = document.querySelectorAll('.eye');
  eyes.forEach((eye) => {
    // Поворот глаза (как раньше)
    const rect = eye.getBoundingClientRect();
    const x = rect.left + rect.width / 2;
    const y = rect.top + rect.height / 2;
    const rad = Math.atan2(event.clientX - x, event.clientY - y);
    const rot = rad * 180 / Math.PI * -1;
    eye.style.transform = `rotate(${rot}deg)`;

    // Случайное смещение зрачка
    const pupil = eye.querySelector('.pupil');
    if (pupil) {
      const maxShift = 15; // максимальное смещение в пикселях
      const shiftX = (Math.random() * 2 - 1) * maxShift;
      const shiftY = (Math.random() * 2 - 1) * maxShift;
      pupil.style.transform = `translate(calc(-50% + ${shiftX}px), calc(-50% + ${shiftY}px))`;
    }
  });
});
```
Теперь при каждом движении мыши зрачки будут немного дрожать, создавая живой эффект.
