### **Лекция-Воркшоп: От статики к магии. Введение в адаптивный и интерактивный веб-дизайн**

**Цель:** Научиться создавать интерактивный, анимированный элемент (flip-карточку) с нуля, понять связь HTML, CSS, JavaScript и освоить базовые принципы декомпозиции кода и отладки.

*   **Фокус на дизайн и повторение:** «Ваша главная задача сегодня — думать, как дизайнер. Как элемент будет выглядеть, как будет вести себя при взаимодействии. Все непонятные моменты из прошлых модулей — задавайте вопросы, мы их повторим в контексте задачи».
*   **Вопрос аудитории:** «Кто уже пробовал делать что-то интерактивное? Что именно?»

**2. Знакомство с ресурсами (5 мин)**
*   Показываю в чате/на слайде:
    *   **Сайт с материалами** (Figma-макет карточки, стартовые файлы).
    *   **Пароль для доступа** (если требуется).
    *   **Пространство в Build** или аналоге (где можно сразу писать код).

*   **Лайфхак:** «Всегда держите записи прошлых модулей под рукой. Повторение — мать не только учения, но и крепкого кода».

**3. Практический старт: от текста к структуре (10 мин)**

### **Часть 2: От истории к современности. Добавляем «магию» (40 минут)**

**1. Мост от jQuery к ванильному JS (5 мин)**
*   **Мини-история:** «Раньше для такой анимации использовали библиотеку jQuery. Это был спасательный круг. Но современный JavaScript (ES6+) и CSS настолько мощны, что чаще всего можно обойтись без неё. Это делает сайты быстрее и проще в поддержке».
*   **Теза:** «Сегодня наша «магия» будет на чистом CSS и JS».

**2. Начало работы с проектом: организуем файлы (10 мин)**
*   **Live-Coding:** Создаю файлы `style.css` и `script.js`. Показываю, как подключить их к HTML (`<link>` и `<script src="..." defer>`).
*   **Важный принцип:** «Разделение ответственности: HTML — структура, CSS — внешний вид, JS — поведение. Это основа чистого кода».
*   **Проверка:** Запускаю страницу в браузере, открываю Инструменты разработчика (F12). Показываю вкладки Elements, Styles, Console. Убеждаемся, что файлы подключены (они видны в Sources/Network).

**3. Создаём Flip-карточку: структура и базовые стили (25 мин)**
*   **Адаптация примера:** Беру готовую структуру flip-карточки (из материалов). Объясняю каждую обёртку:
    ```html
    <div class="flip-card">
      <div class="flip-card-inner">
        <div class="flip-card-front"> <img src="cat.jpg" alt="Кот"> </div>
        <div class="flip-card-back"> <h3>Секрет!</h3> </div>
      </div>
    </div>
    ```
*   **Объясняю:** `flip-card` — статичная рамка с `perspective`. `flip-card-inner` — движущаяся часть. `front`/`back` — стороны.
*   **Live-Coding (CSS):** Пишу базовые стили: размеры, `border`, `background`. Использую `width: 100%` для картинки. **Важно:** сразу добавляю комментарии в CSS (`/* === CARD STYLES === */`).
*   **Первые 3D-эффекты:**
    *   `perspective: 1000px;` — «Глубина сцены. Чем меньше значение, тем более выражен 3D-эффект».
    *   `transform-style: preserve-3d;` — «Чтобы дочерние элементы жили в одном 3D-пространстве».
    *   `position: absolute/relative` — «Чтобы стороны карточки накладывались друг на друга».
*   **Задание студентам:** «Скопируйте структуру, подключите свою картинку, настройте базовые размеры под свой дизайн».

### **Часть 3: Оживляем карточку. JavaScript и продвинутый CSS (60 минут)**

**1. Анимация переворота на CSS (15 мин)**
*   **Ключевое правило:** `transform: rotateY(180deg);` — «Это поворот на 180 градусов вокруг вертикальной оси».
*   **Секретное свойство:** `backface-visibility: hidden;` — «Скрывает «изнанку» элемента. Без него мы увидим заднюю сторону сразу».
*   **Создаём класс-триггер:** `.flip-card-inner.flipped { transform: rotateY(180deg); }` — «Это состояние «перевёрнуто».
*   **Добавляем переход:** `transition: transform 0.8s;` — «Плавность анимации».
*   **Проверяем:** Вручную добавляю класс `flipped` к `flip-card-inner` в Инспекторе браузера. Карточка должна анимированно перевернуться.

**2. «Включаем» анимацию кликом с помощью JavaScript (25 мин)**
*   **Введение в функцию:** «Функция — это инструкция, которая выполняется по команде».
    ```javascript
    function flipCard() {
      console.log('Карточка хочет перевернуться!'); // Проверка
      this.classList.toggle('flipped'); // this - это элемент, на который кликнули
    }
    ```
*   **Поиск элемента и подписка на событие:**
    ```javascript
    const flipCardElement = document.querySelector('.flip-card-inner');
    flipCardElement.addEventListener('click', flipCard);
    ```
*   **Принцип «слушателя» (EventListener):** «Мы говорим браузеру: «Эй, как только на этот элемент кликнут, выполни эту функцию»».
*   **Обязательная проверка в Console:** Если что-то не работает, ищем ошибки (красный текст) и проверяем, выводится ли наш `console.log`.

**3. Усложняем: счётчик кликов и условная анимация (20 мин)**
*   **Идея:** «А что если карточка будет переворачиваться не сразу, а только после 3-х кликов?»
*   **Вводим переменную-счётчик:** `let clickCount = 0;` вне функции.
*   **Меняем логику функции `flipCard`:**
    ```javascript
    function flipCard() {
      clickCount++; // Увеличиваем счётчик на 1. То же, что clickCount = clickCount + 1
      console.log(`Кликов: ${clickCount}`);

      if (clickCount >= 3) {
        this.classList.toggle('flipped');
        clickCount = 0; // Сброс для следующего раза
      }
    }
    ```
*   **Объясняю оператор `%` (остаток от деления):** «А если мы хотим переворот каждые 3 клика? Тогда условие будет `if (clickCount % 3 === 0)`. Это значит «если клик делится на 3 без остатка»».
*   **Демонстрация и эксперимент:** Меняем числа, смотрим на поведение.

### **Часть 4: Декомпозиция и отладка — навыки профессионала (30 минут)**

**1. Рефакторинг: делим код на функциональные области (15 мин)**
*   **Проблема:** «Представьте, что у вас 10 таких карточек и ещё 5 анимаций. Код в одном файле станет нечитаемым».
*   **Решение:** Показываю, как вынести функцию создания карточки в отдельный модуль `cardBuilder.js`, а работу со счётчиками — в `uiLogic.js`.
*   **Принцип:** «Одна функция — одна задача. Один файл — одна логическая область».

**2. Работа с ошибками (15 мин)**
*   **Имитирую классические ошибки:**
    1.  Опечатка в имени файла: `<script src="scrypt.js"></script>`. Консоль выдаст **404 (Not Found)**.
    2.  Ошибка в селекторе: `document.querySelector('.flip-card-innner')`. Консоль выдаст `null`, а при попытке добавить `EventListener` — ошибку.
    3.  Опечатка в свойстве CSS: `backface-visibility: hiddeen;`. Стиль не применится.
*   **Алгоритм отладки:**
    1.  **Открой Консоль (F12).** Ищи красные ошибки.
    2.  **Проверь Sources.** Все ли файлы загрузились?
    3.  **Используй `console.log`.** Дошёл ли код до этой строки? Чему равна переменная?
    4.  **Смотри в Инспектор.** Применились ли стили? Есть ли нужные классы?
