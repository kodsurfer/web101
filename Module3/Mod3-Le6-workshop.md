

# Воркшоп по интерактивному дизайну для веб-дизайнеров
## 👀 Создаём живые интерфейсы: глаза, следящие за курсором

### Зачем это нужно?

Вы наверняка видели сайты, где элементы интерфейса реагируют на движение мыши: параллакс, кнопки с «подсветкой», забавные персонажи с двигающимися глазами. Такие мелочи делают дизайн живым, запоминающимся и вовлекающим. Сегодня мы научимся создавать подобные эффекты своими руками.

**Что мы сделаем:**
- Напишем код для двух глаз, которые будут следить за курсором.
- Разберёмся, как браузер «видит» движения мыши.
- Поиграем с координатами, углами и CSS-анимацией.
- В финале каждый сможет добавить что-то своё: цветные зрачки, несколько пар глаз, случайные эмоции.

**Важно:** мы не просто копируем код, а учимся мыслить как разработчики интерфейсов. Даже если вы не планируете становиться программистом, понимание этих механизмов поможет вам увереннее общаться с разработчиками и создавать более реалистичные макеты.

## 1. Подготовка рабочего места

Создайте на компьютере папку `eyes-follower`. Внутри неё три файла:

- `index.html` — скелет страницы
- `style.css` — внешность наших глаз
- `script.js` — их поведение

Подключите CSS и JS к HTML (преподаватель показывает, как это делается). Проверьте, что всё работает: например, добавьте в `script.js` команду `console.log('Всё готово!');` и откройте консоль браузера (F12 → Console). Увидели сообщение? Отлично, можно двигаться дальше.

## 2. Рисуем глаза в коде (HTML + CSS)

### 2.1. Базовая разметка

В `index.html` внутри `<body>` создаём секцию, которая займёт весь экран, и контейнер для глаз:

```html
<section class="scene">
  <div class="eyes-wrapper">
    <!-- здесь появятся глаза -->
  </div>
</section>
```

**Почему именно так?** Секция `.scene` будет фоном и областью слежения, а `eyes-wrapper` поможет расположить глаза по центру.

### 2.2. Добавляем глаза (быстро с Emmet)

Внутри `eyes-wrapper` с помощью Emmet (пишем `.eye*2` и нажимаем Tab) создаём два `<div class="eye"></div>`. Теперь в разметке два блока для будущих глаз.

### 2.3. Оформляем в CSS

В `style.css` напишем обнуление отступов и зададим стили для сцены:

```css
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

.scene {
  width: 100vw;
  height: 100vh;
  display: flex;
  justify-content: center;
  align-items: center;
  background: #f0f4fa; /* нежный фон */
}

.eyes-wrapper {
  display: flex;
  gap: 30px; /* расстояние между глазами */
}
```

Теперь сами глаза:

```css
.eye {
  width: 100px;
  height: 100px;
  background: white;
  border-radius: 50%;
  position: relative;
  box-shadow: 0 5px 15px rgba(0,0,0,0.2);
}
```

Пока это просто белые круги. Добавим зрачки через псевдоэлемент `::after` (или `::before`):

```css
.eye::after {
  content: '';
  position: absolute;
  width: 30px;
  height: 30px;
  background: #2c3e50;
  border-radius: 50%;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

Теперь у нас симпатичные глаза с чёрными зрачками. Откройте `index.html` в браузере — глаза должны быть по центру экрана.

## 3. Оживляем: математика движения

### 3.1. Слушаем мышь

Чтобы глаза реагировали, нам нужно знать, где находится курсор. В JavaScript за это отвечает событие `mousemove`. Добавим в `script.js`:

```javascript
document.addEventListener('mousemove', function(event) {
  console.log('X:', event.clientX, 'Y:', event.clientY);
});
```

Подвигайте мышью и посмотрите в консоль — числа меняются. Это координаты курсора относительно окна браузера.

### 3.2. Находим центры глаз

Теперь для каждого глаза нужно определить его центр. Координаты элементов можно получить через свойства `offsetLeft` и `offsetTop`. Но проще использовать метод `getBoundingClientRect()`, который даёт точные координаты относительно окна.

```javascript
const eyes = document.querySelectorAll('.eye');

document.addEventListener('mousemove', function(event) {
  eyes.forEach(eye => {
    const rect = eye.getBoundingClientRect();
    const eyeX = rect.left + rect.width / 2;
    const eyeY = rect.top + rect.height / 2;
    console.log('Центр глаза:', eyeX, eyeY);
  });
});
```

### 3.3. Угол поворота: главная хитрость

Представьте, что центр глаза — это точка на плоскости, а курсор — другая точка. Нам нужно, чтобы зрачок «смотрел» в сторону курсора. Для этого вычислим угол между горизонтальной осью и вектором, соединяющим центр глаза с курсором.

В математике это делает функция `Math.atan2(dy, dx)`. Она возвращает угол в радианах. Переведём радианы в градусы, так как CSS работает с градусами.

```javascript
document.addEventListener('mousemove', function(event) {
  eyes.forEach(eye => {
    const rect = eye.getBoundingClientRect();
    const eyeX = rect.left + rect.width / 2;
    const eyeY = rect.top + rect.height / 2;

    const dx = event.clientX - eyeX;
    const dy = event.clientY - eyeY;

    const rad = Math.atan2(dy, dx);
    const deg = rad * (180 / Math.PI);

    // Применяем поворот к зрачку (псевдоэлементу)
    eye.style.transform = `rotate(${deg}deg)`;
  });
});
```

**Важно:** мы поворачиваем весь глаз, а не только зрачок. Из-за этого зрачок будет двигаться по кругу, оставаясь на месте. Чтобы зрачок реально смещался, нужна другая логика (мы обсудим её позже). Но для начала и такой эффект выглядит забавно.

Попробуйте! Зрачки крутятся, но остаются в центре — похоже на вращающиеся глаза. Если хотите, чтобы зрачок двигался линейно, нужно менять не `rotate`, а свойства `top` и `left`. Но это сложнее, поэтому для первой версии оставим как есть.

### 3.4. Плавность и ограничения

Событие `mousemove` срабатывает очень часто. Это может нагружать браузер. Для плавности можно использовать `requestAnimationFrame`, но пока обойдёмся без него.

Заметили, что зрачок может «выворачиваться» неестественно? Это потому, что мы вращаем весь глаз. В конце воркшопа обсудим, как улучшить модель.

## 4. Добавляем обратную связь

Сделаем так, чтобы на странице отображались текущие координаты курсора и информация о том, с какой он стороны.

### 4.1. HTML для данных

Добавим в `index.html` после сцены блок:

```html
<div class="info">
  <p>X: <span id="x-coord">0</span></p>
  <p>Y: <span id="y-coord">0</span></p>
  <p>Сторона: <span id="side">центр</span></p>
</div>
```

И немного стилей, чтобы блок не мешал:

```css
.info {
  position: fixed;
  bottom: 20px;
  left: 20px;
  background: rgba(0,0,0,0.7);
  color: white;
  padding: 10px 20px;
  border-radius: 30px;
  font-family: sans-serif;
  font-size: 14px;
}
```

### 4.2. Обновляем информацию

В обработчике `mousemove` будем обновлять текст в спанах:

```javascript
const xSpan = document.getElementById('x-coord');
const ySpan = document.getElementById('y-coord');
const sideSpan = document.getElementById('side');

document.addEventListener('mousemove', function(event) {
  const mouseX = event.clientX;
  const mouseY = event.clientY;

  xSpan.textContent = mouseX;
  ySpan.textContent = mouseY;

  const centerX = window.innerWidth / 2;
  if (mouseX < centerX) {
    sideSpan.textContent = 'слева';
  } else if (mouseX > centerX) {
    sideSpan.textContent = 'справа';
  } else {
    sideSpan.textContent = 'точно по центру';
  }

  // ... остальной код с глазами
});
```

Теперь пользователь видит, как мышь взаимодействует с экраном. Это не только полезно для отладки, но и просто интересно.

## 5. Творческое задание: случайные эмоции

Интерактивность — это не только слежение. Давайте добавим элемент неожиданности: кнопка, которая меняет «настроение» глаз.

### 5.1. Разметка

Добавим кнопку и блок для вывода случайного слова:

```html
<div class="randomizer">
  <button id="mood-btn">Какое у меня настроение?</button>
  <p id="mood-output">😐</p>
</div>
```

### 5.2. Логика

В JavaScript создадим массив слов-эмоций:

```javascript
const moods = ['😊 довольный', '😴 сонный', '🤔 задумчивый', '😡 сердитый', '🥳 весёлый'];
const btn = document.getElementById('mood-btn');
const output = document.getElementById('mood-output');

btn.addEventListener('click', () => {
  const randomIndex = Math.floor(Math.random() * moods.length);
  output.textContent = moods[randomIndex];
});
```

Кликните по кнопке — каждый раз появляется новое настроение. А теперь — задание со звёздочкой: попробуйте сделать так, чтобы при смене настроения менялся цвет зрачков или форма глаз (например, через добавление CSS-классов). Это уже ваша творческая часть.

## 6. Итоги и идеи для развития

### Что мы сегодня освоили
- Событие `mousemove` и координаты курсора.
- Как получить координаты элемента на странице.
- Простую тригонометрию для расчёта угла.

### Куда двигаться дальше
- **Настоящее смещение зрачка.** Вместо поворота всего глаза можно вычислять смещение по X и Y с ограничением радиуса. Это хороший вызов.
- **Несколько пар глаз.** Добавьте ещё несколько глаз на страницу — например, улыбающееся лицо или группу персонажей.
- **Анимация при клике.** Пусть глаза «моргают», когда пользователь кликает.
- **Интеграция с веб-камерой.** Можно использовать библиотеку `tracking.js` для слежения за лицом.
