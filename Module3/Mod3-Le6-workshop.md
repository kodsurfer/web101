# Интерактивные интерфейсы на CSS и JavaScript
«Глаза, следящие за курсором»

---

## Часть 1. Введение и организация проекта

### 1.1. Цели занятия
- Научиться отслеживать положение курсора и координаты мыши в браузере.
- Познакомиться с CSS-свойствами для позиционирования и анимации.
- Создать интерактивный элемент: глаза, следящие за движением мыши.
- Понять, как связывать логику JavaScript с визуальными изменениями CSS.
- В конце воркшопа — обсуждение макетов, альтернативных идей и консультация по проектам студентов.

### 1.2. Структура занятия
1. **Теоретическая часть** — основные понятия: события мыши, координаты, псевдоэлементы, трансформации.
2. **Практика** — поэтапное создание проекта (глаза + дополнительные эффекты).
3. **Заключительная часть** — показ результатов, обсуждение, идеи для развития, консультации.

### 1.3. Организация рабочего пространства
- Создаём папку проекта, например `mouse-interaction`.
- Внутри создаём файлы: `index.html`, `css/all.css`, `js/script.js`.
- Подключаем CSS через `<link>` в `<head>`, а JavaScript — через `<script>` перед закрывающим тегом `</body>`.
- Обязательно проверяем подключение (например, выводим `console.log('Script connected')`).

## Часть 2. Создание базовой структуры глаз (HTML/CSS)

### 2.1. HTML-каркас
В `index.html` внутри `<body>` создаём секцию с классом `mouse-cursor` (для отслеживания движений мыши). Внутри неё — контейнер для глаз.

```html
<section class="mouse-cursor">
  <div class="container">
    <!-- глаза будут здесь -->
  </div>
</section>
```

### 2.2. Быстрое создание элементов
- Используем эммет-синтаксис: в контейнере пишем `.eye*2` и табуляция — создадутся два `div` с классом `eye`.
- Дополнительно можно добавить обёртку для глаз с классом `eyes-wrapper` для удобного позиционирования.

### 2.3. Базовая стилизация глаз (CSS)
В `all.css` сбрасываем внешние отступы и задаём `box-sizing: border-box` для всех элементов:

```css
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}
```

- Глаза: ширина 30px, высота 30px, фон `lightgray`, скругление 100%.
- Контейнер для глаз: ширина 70px, flex, `justify-content: space-between`, чтобы глаза стояли рядом с отступом.

### 2.4. Создание зрачков через псевдоэлементы
- У каждого глаза (`position: relative`) создаём псевдоэлемент `::before` (или `::after`) с `content: ""`, `position: absolute`, шириной 12px, высотой 12px, фоном чёрным, скруглением 100%.
- Центрируем зрачок внутри глаза: `left: 50%; top: 50%; transform: translate(-50%, -50%);` (вместо ручного расчёта отступов — более гибкий способ).

### 2.5. Центрирование всей секции на экране
- Секции с классом `mouse-cursor` задаём `width: 100vw; height: 100vh; display: flex; justify-content: center; align-items: center;`
- Проверяем результат в браузере — два серых кружка с чёрными зрачками по центру экрана.

## Часть 3. Оживляем глаза: JavaScript и математика

### 3.1. Основы событий мыши
- Объясняем, что такое событие `mousemove`.
- Показываем, как добавить слушатель события на весь документ:

```javascript
document.addEventListener('mousemove', function(event) {
  console.log('Mouse X:', event.pageX, 'Mouse Y:', event.pageY);
});
```

### 3.2. Находим центры глаз
- Получаем все элементы с классом `eye` через `document.querySelectorAll('.eye')`.
- Для каждого глаза вычисляем его центр относительно документа:  
  `const eyeCenterX = eye.offsetLeft + eye.offsetWidth / 2;`  
  `const eyeCenterY = eye.offsetTop + eye.offsetHeight / 2;`
- Выводим координаты в консоль для проверки.

### 3.3. Вычисление угла поворота зрачка
- Используем метод `Math.atan2(dy, dx)`, где `dx = mouseX - eyeCenterX`, `dy = mouseY - eyeCenterY`.
- Полученный угол в радианах переводим в градусы: `angle = Math.atan2(dy, dx) * 180 / Math.PI`.
- Применяем поворот к зрачку через CSS-свойство `transform: rotate(angle deg)`.

```javascript
eye.style.transform = `rotate(${angle}deg)`;
```

**Важный нюанс:** зрачок должен быть смещён относительно центра глаза. Если мы поворачиваем весь глаз, то зрачок будет двигаться по кругу, а не линейно. Можно вместо поворота использовать трансформацию смещения, но для простоты используем rotate с дополнительным смещением или позиционирование зрачка по вектору. Обсуждаем с группой разные подходы.

> **Подсказка:** проще реализовать движение зрачка как смещение по осям X и Y в пределах глаза. Для этого нужно вычислить нормализованный вектор направления и умножить на радиус допустимого смещения. Это хорошая задача для продвинутых студентов.

### 3.4. Демонстрация результата
- Проверяем: при движении мыши зрачки «смотрят» в сторону курсора.
- Если зрачки выходят за пределы глаза — обсуждаем ограничивающую логику (например, через `Math.min/max`).

### 3.5. Дополнительные примеры работы с координатами
- Показываем, как выводить координаты курсора прямо на страницу (создаём два `<span>` и обновляем их `innerHTML`).
- Определяем, слева или справа от центра экрана находится курсор, и выводим статус.
- Привязываем анимацию вращения к этому статусу (меняем класс `right`/`left` на контейнере).

```javascript
if (mouseX < window.innerWidth / 2) {
  statusSpan.innerHTML = 'Слева';
  container.classList.add('left');
  container.classList.remove('right');
} else {
  statusSpan.innerHTML = 'Справа';
  container.classList.add('right');
  container.classList.remove('left');
}
```

В CSS для классов `.left` и `.right` определяем разную анимацию (например, вращение в разные стороны).

## Часть 4. Творческое задание: рандомайзер слов

### 4.1. Постановка задачи
- Создаём кнопку с тремя вопросительными знаками.
- При клике на кнопку в специальном блоке появляется случайное слово из заранее подготовленного списка.
- Стилизуем кнопку и блок, добавляем эффекты наведения.

### 4.2. Реализация
- В HTML добавляем секцию `randomizer` с кнопкой и параграфом для вывода.
- В JS создаём массив слов: `const words = ['добрый', 'злой', 'готов к сессии', 'креативный', ...]`.
- Вешаем обработчик клика на кнопку.
- В обработчике генерируем случайный индекс: `Math.floor(Math.random() * words.length)`.
- Выводим слово в параграф.

### 4.3. Потенциал развития
- Вместо слов можно использовать классы и менять стили случайным образом.
- Можно анимировать появление нового слова.

## Материалы для преподавателя

### Ключевые понятия, которые нужно объяснить просто:
- **События** — это действия пользователя, на которые браузер может реагировать.
- **Координаты мыши** — `event.pageX` и `event.pageY` дают положение относительно документа.
- **offsetLeft/offsetTop** — позиция элемента относительно родителя.
- **Math.atan2** — функция, возвращающая угол между осью X и вектором (dy, dx). Полезна для направления.
- **transform: rotate** — поворачивает элемент на заданный угол.

### Типичные ошибки и их предотвращение:
- Забыли указать единицы измерения (`deg`).
- Не учли, что `offsetLeft` может считаться относительно ближайшего позиционированного родителя — нужно проверить, что родитель не имеет `position: relative` без необходимости.
- Зрачки «убегают» за пределы глаза — нужно ограничить смещение максимальным радиусом.

### Рефакторинг кода:
- Вынести функции отдельно: `getEyeCenters()`, `updateEyePosition(event)`.
- Использовать `requestAnimationFrame` для оптимизации производительности при `mousemove`.
