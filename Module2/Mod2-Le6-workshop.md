### **Воркшоп: «Динамический веб: от бегущей строки до сложных анимаций»**

**Целевая аудитория:** Студенты веб-дизайнеры (начальный/средний уровень верстки).
**Формат:** 2 пары (теоретическая + практическая с консультацией).
**Инструменты:** HTML, CSS, редактор кода (VS Code), браузер.

---

### **ЧАСТЬ 1: ТЕОРИЯ И СТАРТ ПРОЕКТА (Первая пара)**

**Тема:** «Принципы CSS-анимации и адаптивной верстки»

**Введение (15 мин)**
*   **Цель воркшопа:** Научиться создавать ключевые типы анимаций (линейные, трансформации, пошаговые) и понимать логику их адаптации под разные экраны.
*   **Обсуждение:** Какие анимации вы видели на сайтах? Какие из них полезны, а какие раздражают? (Цель: понять UX-роль анимации).
*   **План:** Первая пара — пишем код вместе. Вторая пара — консультация по вашим проектам (можно без готового кода, только с макетом и идеями).

**1. Начало работы: база проекта (15 мин)**
*   **Практика:** Создаем папку проекта, два файла: `index.html` и `style.css`.
*   **Связывание:** Прописываем `<link>` в HTML.
*   **Стандартная обвязка HTML:** `<!DOCTYPE html>`, структура `html`, `head`, `body`.
*   **Базовые стили (Reset):**
    ```css
    * {
        margin: 0;
        padding: 0;
        box-sizing: border-box;
    }
    ```
    *Объясняем, зачем это нужно: обнуление отступов браузера, и `box-sizing: border-box` для удобного расчета размеров (padding и border включаются в width/height).*

**2. Наша первая анимация: бегущая строка (45 мин)**
*   **Шаг 1: Контейнер.**
    *   Создаем `div.container-marquee`.
    *   Внутри него `div.marquee-content`.
    *   *Лайфхак:* Используем эммет (расширение в VS Code): `.container-marquee>.marquee-content`.
*   **Шаг 2: Стилизация контейнера и текста.**
    *   Контейнеру задаем `width: 100vw`, `height: 10vh`, `background-color: #333`.
    *   Тексту (внутри `.marquee-content`) задаем `font-size: 4vw`, `color: white`, `font-family: sans-serif`, `text-transform: uppercase`.
    *   *Ключевой момент:* Используем `vw` для адаптивности. Обсуждаем, как `vw/vh` зависят от размера окна браузера.
*   **Шаг 3: Позиционирование и дублирование контента.**
    *   Контейнеру `.container-marquee` задаем `position: relative; overflow: hidden;`.
    *   Элементу `.marquee-content` задаем `position: absolute; white-space: nowrap;`.
    *   **Главная проблема бегущей строки:** Чтобы анимация была непрерывной, контента должно быть в 2 раза больше ширины области видимости. Дублируем текст внутри `.marquee-content` (вручную или через JS, но мы сделаем вручную). Самому элементу задаем `width: 200%;`.
*   **Шаг 4: Создание анимации движения.**
    ```css
    @keyframes run {
        0% { transform: translateX(0%); }
        100% { transform: translateX(-50%); } /* Сдвигаем на половину своей ширины */
    }
    .marquee-content {
        animation: run 10s linear infinite;
    }
    ```
    *   **Объясняем логику:** Ширина контента 200%. Сдвиг на -50% сдвигает его ровно на один "экран". Так создается эффект бесконечного seamless-повтора.
*   **Проверка и выводы:** Запускаем, смотрим. Меняем размер окна браузера — анимация адаптируется. **Ключевой принцип:** Для бесконечного линейного движения нужен дублированный контент шириной 200% и сдвиг на 50%.

**Перерыв 10 мин**

**3. Анимация-трансформация: летающий круг (30 мин)**
*   **Шаг 1: Создаем контейнер.** `div.container-circle`. Задаем размеры (50vw x 40vh), `border-radius: 50%`, `border: 2px dashed #ccc`, `position: relative`.
*   **Шаг 2: Создаем элемент для анимации.** Внутри создаем `div.circle`. Стилизуем его (размеры, цвет, `border-radius: 50%`, `position: absolute`).
*   **Шаг 3: Анимация по траектории.**
    ```css
    @keyframes fly {
        0% { transform: translate(0%, 0%); }
        25% { transform: translate(100%, 50%); }
        50% { transform: translate(40%, 100%); }
        75% { transform: translate(0%, 60%); }
        100% { transform: translate(0%, 0%); }
    }
    .circle {
        animation: fly 8s linear infinite;
    }
    ```
    *   Объясняем: `translate(X%, Y%)` — движение в процентах от размеров **самого элемента**. Для движения в процентах от родителя нужна другая техника (чаще используют `left`/`top` с `%`, но `transform` более производительна).
*   **Шаг 4: Интерактивность.**
    ```css
    .circle:hover {
        animation-play-state: paused;
        background-color: red;
        transition: background-color 0.5s ease;
    }
    ```
    *   Показываем управление анимацией (`paused/running`) и плавное изменение свойств через `transition`.

**Домашнее задание на первую пару:** Создать макет (в Figma или на бумаге) для своего проекта, где будут использованы минимум 2 типа анимаций, изученных сегодня. Продумать адаптивность.

### **ЧАСТЬ 2: ПРАКТИКА И КОНСУЛЬТАЦИЯ (Вторая пара)**

**Тема:** «Разбор полетов: от макета к коду»

**1. Разбор сложной анимации: CSS-секундомер (30 мин)**
*   **Задача:** Создать анимированные цифры, имитирующие работу секундомера (минуты:секунды).
*   **Принцип:** Используем `overflow: hidden` и анимацию `translateY` для длинного столбца цифр.
*   **Структура HTML:**
    ```html
    <div class="stopwatch">
        <div class="digit tens-min">...012345...</div>
        <div class="digit ones-min">...0123456789...</div>
        <div class="divider">:</div>
        <div class="digit tens-sec">...012345...</div>
        <div class="digit ones-sec">...0123456789...</div>
    </div>
    ```
*   **Ключевая техника - шаговая анимация (steps):**
    ```css
    @keyframes tick-ones {
        0% { transform: translateY(0); }
        100% { transform: translateY(-1000%); } /* Сдвигаем на 10 цифр */
    }
    .ones-sec {
        animation: tick-ones 10s steps(10) infinite;
    }
    ```
    *   **Объясняем:** `steps(10)` разбивает плавную анимацию на 10 резких шагов, что создает эффект переключения цифр. Для десятков секунд анимация будет длиться 60 секунд и иметь `steps(6)`.

**2. Консультационный блок (60 мин)**
Работа в формате «свободный микрофон». Студенты задают вопросы по своим проектам. Преподаватель на живых примерах разбирает проблемы.

**Типичные проблемы и их решения (на основе вашего конспекта):**

*   **Проблема:** «Элементы накладываются друг на друга / уезжают».
    *   **Решение:** Проверить `position: relative` у родителя и `position: absolute` у ребенка. Проверить `z-index`. Убедиться, что заданы `width`/`height`.

*   **Проблема:** «Анимация дергается в начале/конце цикла».
    *   **Решение (для бегущей строки):** Убедиться, что ширина контента 200%, а сдвиг в `@keyframes` составляет 50%. Контент должен быть идеально дублирован.

*   **Проблема:** «Как перевести пиксели из макета Figma в `vw/vh`?»
    *   **Решение-формула:** `(Размер в пикселях / Ширина макета в пикселях) * 100 = Размер в vw`.
    *   **Пример:** Элемент шириной 342px на макете 1920px. `(342 / 1920) * 100 ≈ 17.8vw`. Для высоты часто считают от `vh`.

*   **Проблема:** «Не работает `background-image` или не видно картинку».
    *   **Решение:** 1) Проверить путь к файлу (`./img/photo.jpg`). 2) Задать элементу размеры (`width`, `height`). 3) Для фона использовать `background-size: cover` или `contain`.

*   **Проблема:** «Как сделать спрайтовую анимацию (например, бегущего человека)?»
    *   **Решение:** Объединить все кадры в одну длинную горизонтальную/вертикальную картинку (спрайт). Использовать анимацию `background-position` с `steps()`.
    ```css
    @keyframes run-sprite {
        from { background-position: 0 0; }
        to { background-position: -1600px 0; } /* Ширина всего спрайта */
    }
    .character {
        animation: run-sprite 0.8s steps(8) infinite; /* 8 кадров в спрайте */
    }
    ```

**3. Финальные советы и архитектура проекта (15 мин)**
*   **Структура CSS-файлов:** Рекомендуемый порядок подключения: `reset.css` -> `fonts.css` -> `common.css` (базовые стили) -> `animations.css` -> `media.css`.
*   **Организация кода:** Пишем все в одном файле для малого проекта, но сразу группируем: сначала все `@keyframes`, затем все стили элементов.
*   **Именование классов:** Используйте БЭМ (`block__element--modifier`) или просто осмысленные имена (`header`, `menu-button`, `menu-button--animated`).
*   **Чек-лист перед сдачей:**
    1.  Анимация работает плавно и без артефактов.
    2.  При изменении размера окна макет не ломается (проверяем на 1920px, 1280px, 768px, 360px).
    3.  Код отформатирован, нет лишних комментариев.
    4.  Все файлы (шрифты, картинки) лежат в правильных папках и корректно подключены.

**Итог воркшопа:** готовые работающие примеры анимаций, понимание принципов их создания и адаптации, а также индивидуальные рекомендации по проектам.
