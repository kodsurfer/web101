### **Воркшоп: Верстка образовательного сайта "Геометрия"**

**Цель воркшопа:** На практическом примере разобрать полный процесс верстки сайта — от создания репозитория до реализации сложных секций макета. Мы научимся работать с GitHub, организовывать структуру проекта, подключать нестандартные шрифты и позиционировать элементы.

**Необходимые инструменты:** Браузер, GitHub Desktop, VS Code, макет в Figma/Photoshop.

---

### **Часть 1: Подготовка и настройка проекта**

**(Слайд 1: Введение)**
*   **Обсуждение:** Верстка — это перевод дизайн-макета в рабочий код. Сегодня мы будем работать с макетом студенческого проекта, посвященного шрифту "Геометрия".
*   **Цель:** Создать полнофункциональный, адаптивный (на перспективу) лендинг.

**1. Создание репозитория на GitHub**
*   *Действие:* Открываем GitHub в браузере. Создаем новый репозиторий.
*   *Название:* `Geometry` (на латинице).
*   *Настройки:* Публичный репозиторий, *не* добавляем README, .gitignore или license.
*   *Объяснение:* "Публичный" значит, что код смогут увидеть все. Это наше "хранилище" в облаке.

**2. Клонирование репозитория на устройство**
*   *Действие:* Открываем GitHub Desktop. Клонируем только что созданный репозиторий на рабочий стол.
*   *Объяснение:* "Клонирование" — это создание локальной копии репозитория на вашем компьютере. Все изменения мы будем делать локально, а затем "пушить" (отправлять) в облако.

**3. Работа с папкой в VS Code**
*   *Действие:* Открываем папку `Geometry` на рабочем столе через VS Code. Создаем новое рабочее пространство.
*   *Лайфхак:* В настройках VS Code включаем отображение скрытых файлов (например, `.git`).

**4. Организация структуры проекта**
*   *Действие:* Создаем папки в корне проекта:
    *   `css/` (для стилей)
    *   `images/` (для изображений)
    *   `fonts/` (для шрифтов)
    *   `js/` (для скриптов, на будущее)
*   *Проверка:* Открываем папку проекта в системном файловом менеджере (Finder/Проводник), чтобы убедиться, что структура понятна.

**5. Создание главной страницы**
*   *Действие:* В корне проекта создаем файл `index.html`.
*   *Базовый код:* Используем Emmet-аббревиатуру `!` или пишем базовую структуру HTML5.
*   *Проверка:* Сохраняем и открываем файл в браузере. Пока видна пустая страница.

### **Часть 2: Подключение стилей и шрифтов**

**6. Подключение основных стилей**
*   *Действие:* В папке `css/` создаем файлы:
    *   `styles.css` (главный файл стилей)
    *   `reset.css` (для обнуления стандартных отступов браузера)
    *   `fonts.css` (для подключения шрифтов)
    *   `layout.css` (для сетки и основных блоков)
*   *Действие:* В `index.html` подключаем `styles.css` через `<link href="css/styles.css" rel="stylesheet">`.
*   *Действие:* В файл `styles.css` с помощью `@import` подключаем остальные CSS-файлы в правильном порядке:
    ```css
    @import url('reset.css');
    @import url('fonts.css');
    @import url('layout.css');
    ```
*   *Проверка:* В `layout.css` задаем `body { background: red; }`, обновляем браузер. Если фон стал красным — все подключено правильно.

**7. Подключение шрифта "Геометрия"**
*   *Подготовка:* Скачиваем шрифт "Геометрия" во всех начертаниях (16 файлов) в форматах WOFF и WOFF2. Копируем их в папку `fonts/`.
*   *Принцип:* Каждое начертание (Regular, Bold, Light, Italic и т.д.) — это отдельный файл и отдельное правило `@font-face`.

**8. Создание описания шрифта (в `fonts.css`)**
*   *Объяснение:* Правило `@font-face` "объясняет" браузеру, как найти и использовать наш шрифт.
*   *Шаблон:*
    ```css
    @font-face {
        font-family: 'Geometry';
        src: url('../fonts/geometry_regular.woff2') format('woff2'),
             url('../fonts/geometry_regular.woff') format('woff');
        font-weight: 400; /* Normal */
        font-style: normal;
    }
    ```
*   *Ключевые моменты:*
    *   `font-family`: задаем имя, которое будем использовать в CSS. Сложные названия в кавычках.
    *   `src`: указываем путь к файлам. Форматы подключаются в порядке предпочтения (современный WOFF2 первый).
    *   `font-weight` и `font-style`: критически важные свойства! Они говорят браузеру, *когда* использовать это начертание.

**9. Подключение всех 16 начертаний**
*   *Действие:* Копируем и вставляем правило `@font-face` 16 раз.
*   *Задача:* Для каждого файла меняем:
    1.  Путь в `src` (`geometry_bold.woff2`...).
    2.  `font-weight` (700 для Bold, 300 для Light, 500 для Medium и т.д.).
    3.  `font-style` (`italic` для курсивных начертаний).
*   *Работа в группе:* "Давайте вместе проверим по документации шрифта, какие значения `font-weight` соответствуют каждому файлу?"
*   *Итог:* В `fonts.css` должно быть 16 правил `@font-face`.

**10. Базовые стили (в `reset.css` и `layout.css`)**
*   *В `reset.css`:*
    ```css
    * {
        box-sizing: border-box; /* Чтобы padding и border не увеличивали ширину элемента */
        margin: 0;
        padding: 0;
    }
    ```
*   *В `layout.css`:*
    ```css
    body {
        width: 1440px; /* По ширине макета */
        margin: 0 auto; /* Выравнивание по центру */
        display: block;
    }
    ```

### **Часть 3: Верстка первой секции (Главный экран)**

**(Слайд 2: Визуальное деление)**
*   *Методика:* Для удобства разработки добавим всем секциям временную обводку.
    ```css
    section { border: 1px dashed #ccc; }
    ```

**11. Структура HTML для первой секции**
*   *Действие:* В `<body>` создаем первую секцию.
    ```html
    <section class="hero">
        <div class="hero__image">
            <img src="images/start-geometry.svg" alt="Геометрия">
        </div>
        <h1 class="hero__title">Геометрия</h1>
        <div class="hero__author">Автор</div>
    </section>
    ```

**12. Позиционирование элементов**
*   *Задача:* Разместить надпись "Геометрия" и "Автор" поверх фоновой иллюстрации.
*   *Решение:* Используем относительное и абсолютное позиционирование.
    ```css
    .hero {
        position: relative;
        height: 900px; /* Высота первого экрана */
    }
    .hero__title {
        position: absolute;
        font-family: 'Geometry', sans-serif;
        font-size: 216px;
        color: white;
        top: 50%;
        left: 50%;
        transform: translate(-50%, -50%);
    }
    .hero__author {
        position: absolute;
        bottom: 20px;
        right: 20px;
        font-family: 'Geometry', sans-serif;
        font-weight: 500;
    }
    ```
*   *Объяснение:* Делаем секцию `.hero` `relative`, чтобы элементы внутри с `absolute` позиционировались относительно нее.

**13. Работа с фоновым изображением**
*   *Действие:* Экспортируем фон из макета как `hero-bg.svg` в папку `images`.
*   *Действие:* Добавляем фон через CSS.
    ```css
    .hero {
        ...
        background: url('../images/hero-bg.svg') center/cover no-repeat;
    }
    ```

### **Часть 4: Верстка сложной секции ("История шрифта")**

**14. Структура HTML для секции "История"**
*   *Действие:* Создаем секцию с тремя колонками.
    ```html
    <section class="history">
        <div class="history__block">
            <div class="history__numbers">01</div>
            <div class="history__image">
                <img src="images/history-image.svg" alt="История">
            </div>
            <div class="history__text">
                <h2>История шрифта</h2>
                <p class="history__paragraph">...длинный текст с <br> переносами...</p>
            </div>
        </div>
        <!-- Второй блок "Структура" будет похожим -->
    </section>
    ```

**15. Стилизация с помощью Flexbox**
*   *Действие:* Используем Flexbox для расположения колонок в строку.
    ```css
    .history__block {
        display: flex;
        align-items: center; /* Выравнивание по вертикали */
        justify-content: space-between; /* Распределение пространства */
        margin-bottom: 100px; /* Отступ между блоками */
    }
    .history__numbers {
        font-family: 'Geometry';
        font-size: 160px;
        font-weight: 500;
    }
    .history__text {
        width: 588px;
        text-align: right; /* Текст прижат к правому краю */
    }
    ```

**16. Детальная настройка отступов и текста**
*   *Работа с макетом:* "Смотрим в макет: какой точный `margin-left` у картинки? Каков `line-height` у параграфа?"
*   *Пример:*
    ```css
    .history__paragraph {
        font-family: 'Geometry';
        font-size: 18px;
        line-height: 1.5;
        margin-top: 20px;
    }
    .big-text {
        font-size: 24px;
        text-decoration: underline;
    }
    ```

**17. Работа с изображениями (трансформация)**
*   *Задача:* Для второго блока нужно перевернуть изображение.
*   *Решение:*
    ```css
    .history__image--rotated img {
        transform: rotate(180deg);
    }
    ```

### **Часть 5: Секция "Рецепт" и завершение**

**18. Верстка секции с текстом поверх изображения**
*   *Структура HTML:*
    ```html
    <section class="recipe">
        <div class="recipe__background">
            <img src="images/background-recipe.svg" alt="Фон рецепта">
        </div>
        <div class="recipe__content">
            <h2 class="recipe__title">Рецепт приготовления</h2>
            <p>...текст рецепта...</p>
        </div>
    </section>
    ```
*   *Стилизация:* Аналогично первой секции, используем `position: relative` для `.recipe` и `position: absolute` для `.recipe__content`, чтобы разместить текст поверх фона.

**19. Финализация и проверка**
*   *Действие:* Убираем временные обводки у секций.
*   *Проверка:* Сравниваем получившуюся страницу с макетом pixel-perfect. Проверяем отступы, шрифты, выравнивание.
*   *Рефакторинг:* Заменяем "жесткие" `margin` на более предсказуемые `padding` там, где это необходимо для лучшей адаптивности в будущем.
*   *Коммит и пуш:* Фиксируем все изменения в GitHub Desktop и отправляем финальную версию в репозиторий.
