## Лекция: Анимации, баннер с размытием и многостраничное меню для веб-форм

**Тема:** Продвинутая анимация полей ввода, создание динамического баннера с CSS‑фильтрами, построение навбара и основы многостраничности  

---

### 1. Введение

На прошлом занятии мы с вами сверстали базовую форму. Сегодня мы её доработаем:

- Добавим анимации для **плейсхолдеров** (текст внутри поля будет «уезжать» вверх при фокусе и оставаться там, если поле заполнено).
- Создадим **фоновый баннер** с крутящимися цветными кругами и эффектом **blur**.
- Построим **навигационное меню** (навбар) с текстовыми ссылками и иконками.
- Разберём, как из одного HTML‑файла сделать **многостраничный сайт**.

Все анимации мы сделаем на чистом CSS, без JavaScript.

---

### 2. Оживляем плейсхолдеры в форме

#### 2.1. Проблема перекрытия инпута псевдоэлементом

В нашей форме каждый input обёрнут в `<span class="placeholder">`. На этом спане висит псевдоэлемент `::before`, который показывает текст подписи (например, «Имя», «Почта»). Изначально этот псевдоэлемент перекрывает поле ввода. Чтобы исправить это, добавим в CSS:

```css
.placeholder input {
  position: relative;
  z-index: 1;
  background-color: transparent;
}
```

- `position: relative` + `z-index: 1` – поднимаем input выше псевдоэлемента.
- `background-color: transparent` – делаем фон инпута прозрачным, чтобы был виден псевдоэлемент под ним.

#### 2.2. Анимация при фокусе

Нужно, чтобы при клике в поле текст плейсхолдера уменьшался и уезжал вверх.  
Используем **псевдокласс `:focus`** и селектор **`:has()`** (проверяем, есть ли внутри спана инпут в фокусе).

```css
.placeholder:has(input:focus)::before {
  font-size: 12px;
  line-height: 34px;      /* было 54px – элемент «подпрыгивает» вверх */
  transition: all 0.2s;
}
```

Чтобы анимация работала плавно в обе стороны, вешаем `transition` и на сам псевдоэлемент:

```css
.placeholder::before {
  transition: all 0.2s;
}
```

#### 2.3. Анимация при заполненном поле

Если пользователь что‑то ввёл, плейсхолдер тоже должен оставаться в «поднятом» состоянии.  
Поможет псевдокласс **`:placeholder-shown`** – он проверяет, показан ли стандартный атрибут `placeholder`.  
Нам нужно, чтобы стили срабатывали, когда инпут **НЕ пуст** (т.е. `:placeholder-shown` отсутствует). Используем `:not()`:

```css
.placeholder:has(input:not(:placeholder-shown))::before {
  font-size: 12px;
  line-height: 34px;
}
```

**Важно:** у каждого инпута должен быть пустой атрибут `placeholder=""`.  
Добавить его можно через поиск/замену в редакторе (Ctrl+F → найти `value` → заменить на `value placeholder=""`).

После этого при вводе текста плейсхолдер остаётся вверху, а при удалении – возвращается на место.

### 3. Создаём подсказки («вопросики») для полей

Рядом с полями «Дата рождения» и «Телефон» у нас есть кружок с вопросом. При наведении на него должна появляться подсказка.

#### 3.1. Разметка и стили для кружка

```html
<div class="circle">?</div>
<div class="text-birth">Эти данные определяют сервисы, доступные для учётной записи.</div>
```

CSS для кружка:

```css
.circle {
  user-select: none;      /* запрещаем выделение текста */
  cursor: pointer;
  /* позиционирование относительно родителя */
  position: absolute;
  z-index: 3;
  right: 10px;
  top: 20px;
}
```

Подсказка (`text-birth`) скрыта (`display: none`) и имеет абсолютное позиционирование.  
При наведении на кружок показываем следующий за ним элемент через селектор `+`:

```css
.circle:hover + .text-birth {
  display: block;
}
```

Таким же образом делаем подсказку для телефона, меняя только класс и координаты.

### 4. Баннер с анимацией и размытием

#### 4.1. Структура баннера

```html
<div class="banner">
  <div class="colors">
    <div class="color"></div>
    <div class="color"></div>
    <div class="color"></div>
    <div class="color"></div>
  </div>
  <h1>Создайте свой Apple ID</h1>
</div>
```

#### 4.2. Стили для баннера

- Сам баннер: `width: 100%; height: 300px; overflow: hidden;` (обрезаем всё, что выходит за границы).
- Контейнер `.colors` – вращающаяся «карусель» из кругов:
  ```css
  .colors {
    width: 400px;
    height: 400px;
    margin: 0 auto;
    margin-top: -50px;   /* центрируем по вертикали */
    display: flex;
    flex-wrap: wrap;
    animation: colors 20s linear infinite;
  }
  ```
- Каждый `.color` – квадрат со скруглением 50% (круг), ширина/высота 200px или 50%.
- Добавляем разные фоновые цвета: красный, синий, золотой, голубой.

#### 4.3. Анимация вращения

```css
@keyframes colors {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}
```

#### 4.4. Эффект размытия (blur)

Применяем **CSS‑фильтр** к контейнеру `.colors`:

```css
.colors {
  filter: blur(150px);
}
```

Свойство `filter` размывает всё содержимое элемента. В результате круги превращаются в плавное цветное «облако», которое медленно вращается.  
Обратите внимание: существует и `backdrop-filter`, но он размывает фон за элементом, а нам нужно размыть сами круги.

#### 4.5. Заголовок поверх баннера

Заголовок `<h1>` позиционируем абсолютно, центрируем, задаём белый цвет и крупный шрифт.

```css
.banner h1 {
  position: absolute;
  top: 110px;
  width: 100%;
  text-align: center;
  font-size: 40px;
  color: white;
}
```

Готово – фон плавно переливается, а надпись остаётся чёткой.

### 5. Навбар (меню навигации)

#### 5.1. Семантическая разметка

Используем тег `<nav>`. Внутри – обёртка `<div class="nav-wrapper">`, а в ней ссылки `<a>`.

```html
<nav>
  <div class="nav-wrapper">
    <a href="#" class="logo"></a>
    <a href="#">Магазин</a>
    <a href="#">Mac</a>
    <a href="#">iPad</a>
    <a href="#">iPhone</a>
    <a href="#">Watch</a>
    <a href="#">AirPods</a>
    <a href="#">ТВ и дом</a>
    <a href="#">Только в Apple</a>
    <a href="#">Аксессуары</a>
    <a href="#">Поддержка</a>
    <a href="#" class="search"></a>
    <a href="#" class="bag"></a>
  </div>
</nav>
```

#### 5.2. Стилизация навбара

- `nav` – ширина 100%, тёмный фон (`#191919`), высота 48px.
- `.nav-wrapper` – ограничиваем ширину (например, `max-width: 1200px`), центрируем через `margin: 0 auto`, делаем флексом: `display: flex; justify-content: space-between; align-items: center; height: 100%;`.
- Ссылки (`a`): убираем подчёркивание (`text-decoration: none`), цвет – белый. Для плавного появления используем `opacity: 0.6` и при `:hover` меняем на `1`. Добавляем `transition: opacity 0.2s`.

#### 5.3. Иконки (логотип, поиск, корзина)

Иконки реализуем через пустые ссылки с классами `logo`, `search`, `bag`. Задаём им фоновое изображение через `background-image`, ширину и высоту.

```css
.logo, .search, .bag {
  height: 100%;
  background-repeat: no-repeat;
  background-position: center;
  background-size: auto 44px;
}
.logo { width: 14px; background-image: url('ссылка_на_лого'); }
.search { width: 15px; background-image: url('ссылка_на_лупу'); background-position: center top; }
.bag { width: 13px; background-image: url('ссылка_на_сумку'); }
```

Обратите внимание: для иконок, которые являются спрайтами (несколько изображений в одном файле), нужно корректировать `background-position`.

### 6. Многостраничность сайта

#### 6.1. Как связать страницы

- Создайте несколько HTML‑файлов: `index.html`, `catalog.html`, `contacts.html` и т.д.
- Вставьте в каждый файл **одно и то же меню** (можно копировать).
- В атрибуте `href` ссылок укажите путь к соответствующей странице. Если все файлы лежат в одной папке, достаточно имени файла:
  ```html
  <a href="index.html">Главная</a>
  <a href="catalog.html">Каталог</a>
  ```

#### 6.2. Подсветка активного пункта меню

Чтобы пользователь понимал, на какой странице он находится, добавьте активному пункту специальный класс (например, `active`) и пропишите для него дополнительный стиль (подчёркивание, жирность и т.п.).

```html
<a href="form.html" class="active">Форма</a>
```

```css
.nav-wrapper a.active {
  text-decoration: underline;
}
```

### 7. Практическое задание (для закрепления)

Создайте **мини‑сайт из четырёх HTML‑страниц**:

1. Главная (`index.html`)
2. О нас (`about.html`)
3. Контакты (`contacts.html`)
4. Форма (`form.html`)

На каждой странице сделайте простое меню из четырёх ссылок, которые ведут на эти страницы.  
Меню должно быть одинаковым везде.  
Проверьте, что по кликам происходит переход между страницами.

*Это задание поможет вам освоить механику многостраничных сайтов без использования серверных языков.*
