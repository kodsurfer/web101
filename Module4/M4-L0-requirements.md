# Цели и задачи модуля

### **Требования по ККК**

- Описание

  Студенты создают многостраничный промо-сайт на заданную тему (сайт своего проекта, сайт который презентует редизайн проекта и т.п.). Сайт прототипируется на уровне варфреймов (wireframes) и дизайн-макета. Сайт проектируется учитывая принципы статической веб-верстки и содержания динамических модулей Javascript. Сайт должен быть адаптирован под все экраны популярных устройств. Проект верстается и выкладывается через GitHub Pages.

- Итоговый проект

  Презентация включающая варфреймы всех страниц сайта, дизайн-макет сайта в ключевых адаптивных состояниях, описание концепции проекта и макеты поведения интерактивных модулей UI Kit.

- Требование к промежуточному просмотру

  Презентация включающая варфреймы всех страниц сайта, дизайн-макет сайта в ключевых адаптивных состояниях, описание концепции проекта и макеты поведения интерактивных модулей UI Kit.

- Требования к итоговому проекту

  1. Понятное отражение структуры сайта, корректность выстроенной коммуникации;
  2. Использование наработок предыдущих модулей;
  3. Соблюдение логики поведения интерактивных элементов;
  4. Грамотная работа с типографикой;
  5. Использование варфрейма;
  6. Графическая выразительность;
  7. Адаптивность верстки, html/css, javascript.



## Требования к промежуточному просмотру

1. Подключен репозиторий для  итогового проекта 4-го модуля, содержащий в себе базовую файловую структуру. Репозиторий публичный, опубликован gihub.pages.

2. Созданы и в правильном порядке внутри style.css подключены все необходимые css-файлы со стилями: 

    reset.css — для сброса базовых стилей,
    
    fonts.css — для подключения шрифтов,
    
    layout.css — для базовых стилей для десктопа,
    
    adaptive.css — для стилей адаптивов,
    
    animation.css — для анимаций.

3. Главная страница названа index.html.

4. Файл style.css корректно подключен к index.html.

5. Все файлы корректно сгруппированы по папкам: css — в папке stylesheets, картинки — в папке images, шрифты — в папке fonts.

6. Папка javascript, содержащая в себе js-файлы, подключенные к html

7. Подготовить псевдокод, который будет описывать желаемые интерактивные взаимодействия и анимации в макете. Макет может быть на уровне вайрфреймов, дизайн не важен, только функционал.

8. Созданы и корректно проименованы все html-страницы проекта:

- названия на латинице в нижнем регистре и без пробелов,

- каждая страница содержит простейшую работающую навигацию, список из тегов a для перехода по другим html-страницам,
