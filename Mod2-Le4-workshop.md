### **Часть 2: Теория — Сердце Анимации: `@keyframes`**

**(Слайд 2: Директива @keyframes)**

**Преподаватель:** "Всё начинается с директивы `@keyframes`. Если `@media` — это наш инструмент для адаптивности, то `@keyframes` — это наша машина времени для анимации. Мы описываем, что должно происходить с элементом в ключевые моменты времени."

*   **Синтаксис:**
    ```css
    @keyframes имя-анимации {
      0% { /* Стили в начале */ }
      50% { /* Стили в середине */ }
      100% { /* Стили в конце */ }
    }
    ```
*   **Важно:** Для каждой новой анимации мы создаём новый набор `@keyframes`.

### **Часть 3: Практика — Оживляем Простой Объект**

**Преподаватель:** "Давайте начнём с малого. Создадим пульсирующий красный круг."

1.  **Создаём элемент в `index.html`:**
    ```html
    <div class="red-circle"></div>
    ```

2.  **Базовая стилизация в `style.css`:**
    ```css
    .red-circle {
      width: 100px;
      height: 100px;
      background-color: red;
      border-radius: 50%; /* Делаем его круглым */
    }
    ```

3.  **Создаём анимацию изменения цвета и формы:**
    ```css
    @keyframes pulse {
      0% {
        background-color: red;
        border-radius: 50%;
      }
      50% {
        background-color: blue;
        border-radius: 0%; /* Превращаем в квадрат */
      }
      100% {
        background-color: red;
        border-radius: 50%;
      }
    }
    ```

4.  **Подключаем анимацию к элементу:**
    ```css
    .red-circle {
      width: 100px;
      height: 100px;
      /* ... остальные стили ... */

      /* Волшебная строка: */
      animation: pulse 6s infinite linear;
    }
    ```
    **Объясняем свойства:**
    *   `pulse` — имя анимации.
    *   `6s` — длительность (6 секунд).
    *   `infinite` — повторение бесконечно.
    *   `linear` — равномерная скорость (без ускорений).

**Результат:** Студенты видят плавно пульсирующий и меняющий форму круг.

### **Часть 4: Спрайтовые Анимации — Бегущий Марио**

**(Слайд 3: Что такое спрайт?)**

**Преподаватель:** "Теперь перейдём к классике игровой индустрии — спрайтам. Это одно изображение, на котором собраны все кадры анимации. Наша задача — с помощью CSS 'прокручивать' это изображение, создавая иллюзию движения."

1.  **Создаём элемент для Марио:**
    ```html
    <div class="mario-run"></div>
    ```

2.  **Изучаем спрайт в Figma/Photoshop:** "Посмотрите на спрайт. Ширина одного кадра — 68px. Всего кадров — 4. Значит, общая ширина спрайта — 272px."

3.  **Базовые стили для Марио:**
    ```css
    .mario-run {
      width: 68px;   /* Ширина одного кадра */
      height: 68px;  /* Высота кадра */
      background-image: url('images/mario-sprite.png');
      margin: 50px auto; /* Центрируем и отступаем сверху */
    }
    ```

4.  **Создаём анимацию движения кадра:**
    ```css
    @keyframes run {
      0%   { background-position: 0 0; }
      100% { background-position: -272px 0; } /* Сдвигаем на всю ширину спрайта */
    }
    ```
    *   **Объясняем `background-position`:** "Мы двигаем фон справа налево, показывая по одному кадру. Отрицательное значение — это сдвиг влево."

5.  **Подключаем анимацию КРИТИЧЕСКИ ВАЖНЫМ образом:**
    ```css
    .mario-run {
      /* ... остальные стили ... */
      animation: run 0.5s infinite steps(1);
    }
    ```
    *   **Объясняем `steps(1)`:** "Это наше секретное оружие. Вместо плавного сдвига (`linear`), анимация будет перескакивать между кадрами мгновенно. `1` — это количество шагов между ключевыми кадрами. Так как у нас ключевые кадры только в начале и конце, `steps(1)` идеально переключает весь спрайт за один шаг."

**Результат:** Марио бежит на месте!

### **Часть 5: Сложная Анимация — Бегущий и Прыгающий Марио**

**Преподаватель:** "Наш Марио бежит, но он хочет двигаться по экрану и прыгать! Для этого нам понадобится структура-обёртка."

1.  **Усложняем HTML-структуру:**
    ```html
    <!-- Контейнер, который будет двигаться -->
    <div class="mario-container">
      <!-- Внутри сам Марио со своей анимацией бега -->
      <div class="mario-run"></div>
      <!-- Скрытый дублёр для прыжка -->
      <div class="mario-jump"></div>
    </div>
    ```

2.  **Анимация движения контейнера:**
    ```css
    @keyframes move {
      0% { left: 0; }
      100% { left: calc(100% - 68px); } /* Останавливаемся у края */
    }

    .mario-container {
      position: relative;
      animation: move 10s infinite linear;
    }
    ```

3.  **Создаём и настраиваем анимацию прыжка:**
    ```css
    @keyframes jump {
      0%, 100% { opacity: 0; } /* В начале и конце прыжок невидим */
      48%, 52% { opacity: 1; }  /* В середине анимации (около 50%) показываем прыжок */
    }

    .mario-jump {
      position: absolute;
      top: 0;
      left: 0;
      width: 68px;
      height: 68px;
      background-image: url('images/mario-sprite.png');
      background-position: -340px 0; /* Покадренно находим кадр прыжка на спрайте */
      opacity: 0; /* Изначально скрыт */
      animation: jump 10s infinite; /* Синхронизируем с движением */
    }
    ```

**Результат:** Марио бежит слева направо, и в середине пути совершает прыжок!

### **Часть 6: По-Кадровая Анимация — Взрыв**

**(Слайд 4: Много-кадровый спрайт)**

**Преподаватель:** "Теперь применим наши знания к более сложному спрайту — взрыву с 32 кадрами (4 ряда по 8 кадров). Здесь нам нужно точно рассчитать проценты."

1.  **Создаём элемент:**
    ```html
    <div class="explosion"></div>
    ```

2.  **Стили и анимация:**
    ```css
    .explosion {
      width: 151px;
      height: 151px;
      background-image: url('images/explosion-sprite.png');
      /* Размер спрайта: 8x4 кадра -> 1208px x 604px */
    }

    @keyframes blast {
      /* 32 кадра. 100% / 32 кадра ≈ 3.125% на кадр. */
      0% { background-position: 0 0; }
      3.125% { background-position: -151px 0; }
      6.25% { background-position: -302px 0; }
      /* ... и так все 8 кадров первого ряда ... */
      25% { background-position: 0 -151px; } /* Переход на второй ряд */
      /* ... продолжаем для всех рядов ... */
      100% { background-position: -1057px -453px; } /* Последний кадр */
    }

    .explosion {
      animation: blast 2s infinite steps(1);
    }
    ```
    *   **Объяснение:** "Мы используем `steps(1)`, чтобы резко переключаться между кадрами, создавая чёткую анимацию."

### **Часть 7: Интерактивность и Финальные Штрихи**

**Преподаватель:** "Анимацию можно сделать интерактивной!"

1.  **Пауза при наведении:**
    ```css
    .explosion:hover {
      animation-play-state: paused;
    }
    ```

2.  **Анимация по наведению (без @keyframes):**
    ```css
    .red-circle {
      transition: transform 0.5s; /* Плавность для трансформации */
    }
    .red-circle:hover {
      transform: scale(1.5) rotate(180deg); /* Увеличиваем и вращаем */
    }
    ```
